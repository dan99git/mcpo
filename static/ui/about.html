OpenHubUI is a comprehensive MCP (Model Context Protocol) management platform designed to solve the integration challenges that users face when adding MCP servers to OpenWebUI.

ğŸ¯ The Problem We Solve

OpenWebUI recently added native MCP support, but only for HTTP Streamable servers. Most MCP servers in the ecosystem are stdio-based, leaving users frustrated and unable to access the tools they need. OpenHubUI bridges this gap by converting ANY MCP server (stdio, SSE, or HTTP Streamable) into a format that works seamlessly with OpenWebUI.

ğŸ—ï¸ Origins & Evolution

OpenHubUI is based on the original MCPO project, but has evolved significantly. Most of the original codebase has been refactored, enhanced, and extended to meet the growing needs of the MCP ecosystem.

The name "OpenHubUI" was chosen as a nod to the main application it serves, but does not reflect any official association or endorsement by the OpenWebUI developers and maintainers. This is an independent project created by the community, for the community.

ğŸ”§ How It Works

Dual Server Architecture

OpenHubUI runs two servers simultaneously:

  • Port 8000 (Admin Server): Handles the management UI, direct MCP tool access via OpenAPI, and server configuration
  • Port 8001 (Streamable Proxy): Converts MCP servers to HTTP Streamable format for OpenWebUI compatibility
MCP Protocol Conversion

The system automatically converts between MCP protocol formats:

  • stdio â†’ HTTP Streamable: Launches stdio MCP servers as subprocesses and proxies their JSON-RPC over HTTP
  • SSE â†’ HTTP Streamable: Connects to remote SSE MCP servers and converts to HTTP Streamable
  • HTTP Streamable: Passes through natively without conversion
Configuration Management

All MCP servers are defined in a single mcpo.json file:

{
  "mcpServers": {
    "time": {
      "command": "uvx",
      "args": ["mcp-server-time"],
      "enabled": true
    }
  }
}

Changes are automatically detected and servers are reloaded without downtime.

Tool State Management

Individual tool enable/disable states are tracked in mcpo_state.json:

{
  "servers": {
    "playwright": {
      "enabled": true,
      "tools": {
        "browser_navigate": true,
        "browser_snapshot": false
      }
    }
  }
}

This allows fine-grained control over which tools are available to OpenWebUI.

ğŸ¤– Self-Management APIs

OpenHubUI provides REST APIs that allow AI models to manage their own MCP servers programmatically:

Configuration Management
GET /_meta/config/content - Get current config
POST /_meta/config/save - Update config & reload
POST /_meta/reload - Reload all servers
Dependency Management
GET /_meta/requirements/content - Get requirements.txt
POST /_meta/requirements/save - Update & install deps
Server Control
GET /_meta/servers - List all servers
POST /_meta/servers/{name}/enable - Enable server
POST /_meta/servers/{name}/tools/{tool}/enable - Enable tool
