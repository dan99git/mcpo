Getting Started

Quick Start

MCPO automatically discovers and exposes MCP tools as HTTP endpoints. Configure your servers in the Configuration tab.

API Endpoints

GET /{server}/docs - OpenAPI documentation
POST /{server}/{tool} - Execute tool
GET /_meta/servers - List all servers

Example Usage

# Get current time
curl -X POST http://localhost:8000/time/get_current_time

# Search with Perplexity
curl -X POST http://localhost:8000/perplexity/search \
  -H "Content-Type: application/json" \
  -d '{"query": "latest AI news"}'

Configuration Guide

Server Configuration

Edit mcpo.json to add or modify MCP servers:

{
  "mcpServers": {
    "my-server": {
      "command": "npx",
      "args": ["-y", "@my/mcp-server"],
      "enabled": true,
      "env": {
        "API_KEY": "your-key-here"
      }
    }
  }
}

Environment Variables

MCPO_PORT - Server port (default: 8000)
MCPO_HOST - Server host (default: 0.0.0.0)
MCPO_API_KEY - Optional API key for authentication

Troubleshooting

Common Issues

Server shows "Disconnected"
Check that the command and arguments are correct. Ensure required packages are installed.
Tools not appearing
Restart the server from the Logs page. Check server logs for connection errors.
API calls failing
Verify the tool is enabled and server is connected. Check request payload matches expected schema.
